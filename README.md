| 一、总述<br><br>（一）选题意义和国内外的研究现状分析及应用现状<br><br>1\. 选题意义<br><br>随着移动互联网与人工智能技术的深度融合，传统电商模式正面临“信息过载”与“交互单一”的双重挑战。消费者对购物体验的要求已从单纯的商品交易转向个性化、智能化与综合服务，而传统电商平台在功能集成及中小商户赋能方面仍存在不足。一方面，用户在海量商品中难以快速定位目标，传统的文本搜索难以满足“以图搜图”的直观需求；另一方面，中小商户因技术门槛高、运营成本受限，往往缺乏自主可控的电商解决方案，制约了其业务拓展。<br><br>本选题旨在设计并实现一个基于Spring Boot的“小铺宝库”智慧电商系统，具有重要的理论与实践意义。首先，探索AI技术在电商场景中的辅助导航应用，针对用户“难以准确描述商品名称”的场景，利用YOLO算法提供“视觉分类入口”，快速定位商品所属的大类，辅助用户缩小检索范围，再结合人工筛选实现精准查找，这种“AI粗筛+人工细选”的模式是对传统文本搜索的有效补充。其次，提升个性化服务水平，利用协同过滤算法对用户行为进行分析，实现千人千面的商品推荐，解决那些销量小但种类繁多的冷门商品难以被用户发现的问题。最后，构建闭环的会员营销体系，通过积分商城、会员日活动等功能模块，增强用户粘性，为中小型商家提供一套低成本、高适配的数字化经营解决方案，助力其实现精准营销与高效管理。<br><br>2\. 国外研究现状<br><br>以Amazon、eBay为代表的国际电商巨头较早将人工智能技术应用于电商领域。在Web电商系统开发方面，Panjaitan B等人深入研究了基于Web的电子商务系统开发，优化了用户交互体验。在计算机视觉领域，Zhang Z等人提出了一种基于改进YOLOv11的轻量级检测模型，通过优化特征提取网络显著提升了检测精度与速度，为电商场景下的商品快速识别提供了理论支撑。此外，Wang B探讨了基于大数据技术的电商个性化推荐策略，验证了数据驱动对提升用户转化率的有效性。总的来说，国外在电商系统的智能化算法研究与底层架构优化方面已取得显著成果，为本课题提供了坚实的理论与技术参考，但针对中小型商家低成本、轻量级的智能化解决方案研究相对较少，且现有系统往往部署复杂，难以普及。<br><br>3\. 国内研究现状<br><br>国内淘宝、京东等平台已全面普及了视觉搜索与智能推荐。在系统架构层面，任建新等设计了基于Java Web的智慧商城系统，验证了模块化开发的可行性。在算法应用上，程铃钫等设计了基于YOLOv5的商品个性化智能推荐系统，实现了视觉特征与推荐算法的融合；徐富萍等的研究进一步证实了个性化推荐算法对用户决策行为有显著的正向影响。目前，基于Spring Boot的微服务架构已成为主流，但在将最新YOLO算法与协同过滤算法深度整合到中小型电商系统的实践中，仍有较大的探索空间。综上所述，国内在电商应用层面的实践已较为成熟，但在将最新一代YOLO算法与Spring Boot微服务架构深度融合，并针对中小商家构建低成本闭环系统方面，仍有广阔的探索与创新空间。<br><br>4\. 应用现状<br><br>目前市面上的电商生态正向智能化、精细化发展。实际应用中，不仅有面向消费者的智能导购系统，还出现了智能化电商仓储系统，极大提升了物流效率；以及电商售卖比较推荐系统，帮助用户进行价格与性能的优选。现有的中小型电商系统多采用通用模板开发，优点是成本低、上线快，但缺点是功能同质化严重，缺乏对图像数据的处理能力，且难以根据用户行为进行精准营销。本课题将YOLO目标检测算法与协同过滤算法集成于Spring Boot架构中，构建“交易+会员+AI”的业务闭环，是当前中小型电商应用开发的重要创新方向，具有较强的实用价值。<br><br>（二）课题的主要内容<br><br>本系统基于B/S架构，服务于用户和管理员两种角色，主要功能模块划分如下：<br><br>用户角色：<br><br>1\. 注册登录模块：支持用户通过手机号或邮箱注册，并提供登录入口。<br><br>2\. 个人信息管理模块：支持用户修改头像、昵称、密码，以及对收货地址进行增删改查。<br><br>3\. 智慧导购模块：集成YOLO官方预训练模型，识别用户上传图片中的通用物体类别，并自动引导至对应的分类列表页，实现视觉辅助检索；同时基于协同过滤算法，为用户提供个性化商品推荐。<br><br>4\. 购物车管理模块：支持用户将商品加入购物车，修改商品数量，勾选商品进行结算。<br><br>5\. 商品交易模块：支持用户提交订单、模拟支付、查看物流状态、确认收货。<br><br>6\. 商品评价模块：支持用户已购商品进行评分和图文评价。<br><br>7\. 售后模块：支持用户对不满意的商品申请售后服务。<br><br>8\. 会员活动模块：支持用户进行每日签到获取积分，参与特定日期的“会员日”折扣活动，以及在积分商城使用积分兑换权益或商品。<br><br>管理员角色：<br><br>1\. 商品管理模块：支持管理员对商品进行发布、编辑、下架操作，管理商品分类信息。<br><br>2\. 库存管理模块：支持管理员查看商品库存状态，设置库存预警阈值，进行补货操作。<br><br>3\. 用户管理模块：支持管理员查看注册用户列表，对违规用户进行冻结或解封。<br><br>4\. 订单管理模块：支持管理员查看待发货订单并录入物流信息<br><br>5\. 售后管理模块：审核并处理用户的退货与退款申请。<br><br>6\. 活动与营销管理模块：支持管理员策划会员日活动，如设置日期，折扣规则，发放优惠券，管理积分商城的可兑换商品。<br><br>7\. 评价管理模块：支持管理员审核用户评价，删除违规内容。<br><br>8\. 数据监控模块：支持管理员通过数据大屏查看销售统计图表，掌握运营状况。<br><br>（三）拟解决的关键问题及创新点<br><br>关键问题：<br><br>（1）视觉直觉与文本检索的断层问题：用户往往能看到物体但叫不出准确学名，传统文本搜索无法响应此类需求，需解决视觉输入到系统分类的映射问题。<br><br>（2）通用模型在电商中的适配性：YOLO官方模型输出的是80类通用标签，其中包含大量非商品标签。如何设计过滤与映射逻辑，将通用AI的推理结果转化为电商系统可用的分类导航指令，是本系统的关键逻辑。<br><br>（3）用户流失率高与推荐冷启动：新用户无历史行为导致推荐系统失效，且缺乏有效的激励机制维持老用户活跃度。<br><br>（4）高并发场景下的数据与业务联动复杂性：在涉及秒杀、会员日等活动时，订单生成、库存扣减与积分计算之间的事务一致性难以保证。<br><br>创新点：<br><br>（1）构建混合检索机制：区别于追求像素级精准搜索的复杂系统，本系统创新性地提出“AI负责找货架，用户负责挑商品”的轻量化交互思路。利用YOLO的通用检测能力快速定位商品大类，解决搜索的“第一公里”难题，后续交由用户进行个性化筛选。这种设计既规避了训练细粒度模型的成本，又实实在在地降低了用户的搜索门槛。<br><br>（2）构建会员营销闭环与多维度推荐：为了解决留存难题，系统设计了“每日签到+会员日+积分兑换”的完整营销逻辑，并结合热门榜单兜底与协同过滤算法，利用积分体系深度绑定用户交易行为，增强了系统的用户粘性。<br><br>（3）多维度数据联动机制：针对业务流程割裂问题，系统构建了高内聚的数据流通网络。将用户评价数据、交易数据与积分权益体系深度融合，例如：确认收货自动触发积分结算，优质评价内容作为协同过滤算法的加权因子反哺推荐系统。这种数据联动打破了功能孤岛，显著提升了系统的业务连贯性与数据价值挖掘能力。 | 二、设计方案或论文撰写提纲：<br><br>（一）功能设计<br><br>1\. 核心模块设计：<br><br>（1）智慧导购模块：针对用户难以准确描述商品品类的痛点，设计了视觉辅助检索功能。除了基础的分类检索外，重点引入YOLO算法构建视觉搜索功能。系统集成YOLO官方模型。当用户上传图片后，后端调用AI接口识别图像中的Objects。系统设计了一个白名单过滤器，仅保留属于电商属性的类别，忽略背景中白名单外的物体。识别结果将直接作为分类过滤条件查询数据库，例如识别出cup，系统自动展示所有杯具类商品。这是一种“模糊检索”策略，旨在帮助用户快速缩小浏览范围。同时，为了解决那些销量小但种类繁多的冷门商品曝光不足的问题，该模块集成了基于协同过滤的推荐算法。系统记录用户的浏览与购买行为，计算用户间的相似度矩阵，将相似用户喜欢的商品推荐给当前用户，从而实现个性化的流量分发。<br><br>（2）交易与会员模块：为了确保交易过程的完整性与可追溯性，设计了严谨的订单流转逻辑，涵盖购物车管理、下单锁库存、支付结算、物流发货、收货评价全流程，保障业务数据闭环。为了解决用户留存难的问题，设计了积分权益体系。将用户的活跃行为与消费行为转化为积分，配合“会员日”活动进行权益回馈，形成“活跃-消费-回馈-复购”的良性循环。<br><br>（3）后台监控与管理模块：设计思路：为了提高商家的运营决策效率，设计了数据可视化大屏。除了基础的商品与订单管理外，系统实时统计销量趋势与热门品类，帮助商家快速调整营销策略。同时设置评价审核机制，维护平台良好的社区环境。<br><br>2\. 业务流程设计<br><br>用户端流程：<br><br>（1）用户登录并获取身份验证令牌。<br><br>（2）用户上传图片调用AI接口，返回识别结果并展示商品列表。<br><br>（3）加入购物车并提交订单（预扣库存）。<br><br>（4）模拟支付成功（更新订单状态、实扣库存）。<br><br>（5）用户确认收货（触发积分发放策略）。<br><br>（6）积分入账，用户在会员中心消耗积分兑换权益。<br><br>管理端流程：<br><br>（1）管理员登录后台。<br><br>（2）商品管理（发布新商品、上架、设置库存）。<br><br>（3）活动策划（创建会员日活动、设置优惠券）。<br><br>（4）订单处理（查看待发货订单、录入物流信息）。<br><br>（5）售后与评价管理（审核退款与处理举报评价）。<br><br>（6）数据监控（查看销售大屏数据）。<br><br>（二）技术实现<br><br>本系统采用基于Spring Boot + Vue的前后端分离架构，通过RESTful API进行通信，并独立部署YOLO服务。<br><br>（1）系统架构设计<br><br>表现层（Vue.js）：采用组件化开发模式，将“商品卡片”、“导航栏”等通用模块封装，提高开发效率。前端负责页面路由与状态管理，减轻后端渲染压力，提供类似原生App的流畅交互体验。<br><br>业务逻辑层（Spring Boot）：作为系统的核心枢纽，负责处理复杂的业务逻辑。使用Spring Boot的自动配置特性，集成MyBatis Plus与Redis，减少样板代码，使开发重心聚焦于电商业务本身。同时考虑到Python在AI领域的生态优势，将YOLO模型封装为独立服务。Java后端通过HTTP请求调用AI服务，实现业务系统与计算密集型任务的解耦，防止因图像处理耗时过长阻塞主业务线程。<br><br>数据访问层（MySQL + Redis）：MySQL利用其ACID特性，确保订单、库存、资金等核心数据的强一致性；引入Redis作为缓存中间件，存储商品库存、用户Token及热点推荐列表，在高并发场景下减轻数据库IO压力，提升系统响应速度。<br><br>（2）技术手段<br><br>Spring Boot生态应用：利用Spring Security结合JWT实现无状态认证，解决传统Session模式在跨域与分布式场景下的局限性；利用Spring的声明式事务管理@Transactional，确保扣减库存与生成订单发生异常时能自动回滚，保证数据不丢失。<br><br>YOLO目标检测技术：利用YOLO算法“单阶段检测”速度快的特性，在用户上传图片的毫秒级时间内完成物体识别与定位，相比传统R-CNN系列算法，更适合Web端的实时交互场景。<br><br>协同过滤算法：采用基于物品的协同过滤Item-based CF，相比User-based CF，在商品数量相对稳定的电商场景下，计算结果更稳定，且能更好地解释推荐理由：“因为您购买了A，所以推荐B”。 |
